# 热负荷计算与预测性空调控制 v5.0

## 功能概述

基于用户要求实现的高级空调控制系统，综合考虑人体产热、设备产热、环境因素和课表信息。

## 核心特性

### 1. 人体产热计算

根据活动状态精确计算人体产热量:

| 活动状态 | 产热功率 | 适用场景 |
|---------|---------|---------|
| 休息 (resting) | 80W/人 | 静坐休息、睡眠 |
| 思考/上课 (thinking) | 100W/人 | 正常上课、办公 |
| 轻度运动 (light_exercise) | 150W/人 | 刚进入教室、课间活动后 |

数据来源: ASHRAE标准、人体代谢研究

### 2. 电子设备产热

检测并计算设备散热量:

| 设备类型 | 功率 | 说明 |
|---------|------|------|
| 笔记本电脑 | 20W | 用户指定规格 |
| 台式电脑 | 80W | 含显示器 |
| 投影仪 | 200W | 教学设备 |

### 3. 环境因素

- **围护结构传热**: 根据室内外温差计算墙体/窗户传热
- **太阳辐射**: 考虑窗户朝向和遮阳情况
- **室外温度**: 支持API获取、传感器读取或手动设置

### 4. 历史趋势分析

- 记录60分钟历史数据
- 分析人数趋势 (上升/下降/稳定)
- 预测10分钟后热负荷

### 5. 课表驱动的预冷/预热

```json
{
  "schedule": {
    "monday": [
      {"start": "08:00", "end": "09:35", "course": "数学", "expected_people": 45}
    ]
  }
}
```

- **提前10分钟**开始预冷
- 根据预期人数调节目标温度
- 支持教室预约管理

## 控制策略

### 决策逻辑

```
1. 检查当前是否有课/预约
   ├── 有课: 根据温度决定是否开空调
   │   └── 高温 → 开空调
   │   └── 适宜 → 仅风扇
   │
   └── 无课: 检查是否即将上课
       ├── 10分钟内上课 + 室温>26.5°C → 提前预冷
       └── 否则 → 节能模式

2. 分析人数趋势
   ├── 上升趋势 + 预计负荷>2000W → 提前制冷
   └── 下降趋势 → 逐步降低功率

3. 下课处理
   └── 人快走光 + 温度适宜 → 仅风扇维持
```

### 温度设定策略

| 预期人数 | 目标温度 | 风速 |
|---------|---------|------|
| > 40人 | 24°C | 高 |
| 20-40人 | 25°C | 中 |
| < 20人 | 26°C | 低 |

## 使用方式

### 运行系统

```bash
python -m classroom_ac.main_v3 --source 0 --outdoor-temp 32.0
```

### 参数说明

```bash
--source            视频源 (0=摄像头, 路径=视频文件)
--demo              演示模式 (不控制实际空调)
--target-temp       目标温度 (默认26°C)
--outdoor-temp      室外温度 (默认30°C)
```

### 运行时快捷键

| 按键 | 功能 |
|------|------|
| q | 退出 |
| t | 调整目标温度 |
| o | 调整室外温度 (模拟) |

## 示例热负荷计算

### 场景1: 正常上课
- 30人, 思考状态, 20台笔记本
- 室外32°C, 室内28°C
- **总热负荷: 5550W** (超过3500W空调容量，需要高功率运行)

### 场景2: 课间休息后
- 30人, 轻度运动后 (刚进入)
- 比思考状态多产热 **+1400W**

### 场景3: 夜间自习
- 10人, 休息状态, 无太阳辐射
- **总热负荷: 1100W** (低负荷，风扇即可)

## 节能效果

通过预冷策略和智能控制，预计可节能:
- 避免上课后才开启空调的高负荷启动
- 根据实际人数调节温度，避免过冷
- 下课及时切换到低功耗模式
